# UX Analysis & Improvement — จัดการเมนู (Menu Management)

> วิเคราะห์จากภาพ `page01.jpg`, `page02.jpg`, `page03.jpg`, `flow.jpg`  
> เป้าหมาย: แก้ปัญหา **"จัดการยาก"** และ **"Scroll เยอะ"** สำหรับเจ้าของร้าน

---

## 1. สรุปปัญหาที่พบจากภาพ

### 1.1 Left Panel — จัดการหมวดหมู่

| ปัญหา | รายละเอียด |
|-------|-----------|
| หมวดหมู่เยอะ = Scroll เยอะ | มี 11+ รายการ (Promotion, Banner, เมนูขายดี, เมนูใหม่, ต้ม, ยอดต, เครื่องเคียง ...) ไม่มี Collapse |
| ไม่รู้ว่าแต่ละหมวดมีเมนูกี่อัน | ไม่แสดง badge count → ไม่รู้ว่าหมวดไหน "ว่าง" หรือ "เต็ม" |
| Drag handle มีแต่ไม่ชัดเจน | icon `⠿` เล็กมาก ผู้ใช้ที่ไม่คุ้นอาจไม่รู้ว่า drag ได้ |
| Search bar เล็กและกลมกลืน | ไม่โดดเด่นพอ ผู้ใช้ข้ามไปไม่ใช้ |

### 1.2 Right Panel — รายการเมนู

| ปัญหา | รายละเอียด |
|-------|-----------|
| Grid 4 คอลัมน์ = Scroll เยอะ | เมนูเยอะ → ต้อง scroll ลงไปเรื่อยๆ ไม่รู้ว่าอีกกี่รายการ |
| ไม่มี count เมนูทั้งหมด | ไม่แสดงว่า "แสดง 8/32 รายการ" |
| Filter tab ซ้ำกับ Category | มี tab "ต้ม, ผัด, แกง, คอด" ขณะที่ Category ซ้ายก็มี "ต้ม" → สับสน |
| ไม่มี Bulk action | ถ้าจะลบ/ปิดหลายเมนูต้องทำทีละอัน |
| ไม่มี Toggle view (Grid/List) | Grid ดูภาพสวย แต่ List ดูได้ทีละเยอะกว่า เหมาะกับการจัดการ |

### 1.3 Form เพิ่ม/แก้ไขเมนู (Drawer)

| ปัญหา | รายละเอียด |
|-------|-----------|
| Form ยาวในหน้าเดียว | ชื่อ + ราคา + ส่วนลด + หมวดหมู่ + ประเภท + รูป + คำอธิบาย = ต้อง scroll ในตัว drawer เอง |
| ทับซ้อน content | Drawer เปิดบัง content หลัก ทำให้ดู context ไม่ออก |

## 2. แนวทางแก้ไข (Recommendations)

---

### 2.1 Left Panel — Collapse + Badge Count ✅ ด่วนที่สุด

**ปัจจุบัน:** รายการหมวดหมู่ยาว scroll ไม่รู้จบ  
**แก้:** Collapsible categories พร้อม badge

```
▼ เมนูขายดี  (12)     ✏ 🗑
  └─ [แสดง sub-menu หรือ expand ด้านล่าง]

▶ เมนูใหม่   (5)      ✏ 🗑
▶ ต้ม         (8)      ✏ 🗑
▶ เครื่องเคียง (3)     ✏ 🗑
```

- **Badge** แสดงจำนวนเมนูในวงกลมสีเทา `(12)` ด้านขวา
- **Collapse all / Expand all** ปุ่มเล็กๆ ด้านบนสุด
- **Default:** เปิดแค่หมวดที่กำลัง active, ที่เหลือ collapse
- Drag handle ขยายให้ใหญ่ขึ้น + tooltip "ลากเพื่อเรียงลำดับ"

---

### 2.2 Right Panel — List View + Pagination ✅ ด่วนที่สุด

**ปัจจุบัน:** Grid 4 คอลัมน์ scroll ยาวมาก  
**แก้:** เพิ่ม Toggle View และ pagination

#### Toggle View (Grid vs List)

```
[🔲 Grid]  [☰ List]                     ค้นหา... 🔍
```

**List View** แสดงได้ ~12-15 รายการในหน้าจอเดียว:
```
┌──────────────────────────────────────────────────────────┐
│ ☐  📷  ยำแสลมอนส           399 บาท   ต้ม  ✅เปิด  ✏ 🗑   │
│ ☐  📷  ข้าวผัดกุ้ง          399 บาท   ผัด  ✅เปิด  ✏ 🗑   │
│ ☐  📷  ข้าวผัดไข่ดาว        149 บาท   ผัด  ✅เปิด  ✏ 🗑   │
└──────────────────────────────────────────────────────────┘
```

#### Pagination หรือ "Load more"
```
แสดง 1-12 จาก 32 รายการ    [< 1 2 3 >]
```
ดีกว่า infinite scroll เพราะเจ้าของร้านต้องการ "รู้ว่ามีเมนูกี่อัน"

---

### 2.3 Bulk Action — จัดการหลายเมนูพร้อมกัน ✅ ลดงานซ้ำซ้อน

**ปัจจุบัน:** ต้องลบ/ปิดทีละอัน  
**แก้:** Checkbox + Bulk action bar

```
☑ เลือก 3 รายการ    [ย้ายหมวดหมู่ ▼]  [ปิดการขาย]  [ลบ]   ✕ ยกเลิก
```

ขั้นตอน:
1. ติ๊ก checkbox ที่เมนูที่ต้องการ
2. Bar ด้านบนขยายโชว์ action ที่ทำได้
3. กดปุ่ม action → confirm dialog → done

---

### 2.4 Quick Edit Inline — แก้ราคาได้เลยโดยไม่ต้องเปิด Drawer

**ปัจจุบัน:** ต้องกด "แก้ไขเมนู" → Drawer เปิด → เลื่อน scroll หาช่อง ราคา  
**แก้:** Double-click ที่ราคา → แก้ inline

```
ยำแสลมอนส
[  399  ] บาท  ← double-click แล้วพิมพ์ราคาใหม่ได้เลย
✅ บันทึก  ✕
```

ใช้กับ: **ราคา** และ **toggle เปิด/ปิดการขาย** เป็นหลัก  
ฟีเจอร์ที่ต้องรายละเอียดเยอะ (รูป, คำอธิบาย) ยังคงใช้ Drawer เหมือนเดิม

---

### 2.5 Form เพิ่มเมนู — แบ่ง Step หรือใช้ Accordion

**ปัจจุบัน:** Form ยาว scroll ใน Drawer  
**ตัวเลือก A: แบ่ง Step (แนะนำ)**

```
Step 1: ข้อมูลพื้นฐาน     Step 2: รูปภาพ     Step 3: หมวดหมู่ & ราคา
  ชื่อเมนู                  อัพโหลดรูป          หมวดหมู่
  คำอธิบาย                  crop/preview        ราคา / ส่วนลด
                                                ประเภทอาหาร
```

ดีเพราะ: เจ้าของร้านรู้ว่าต้องทำอะไรต่อ ไม่รู้สึกว่า form "ยาว"

**ตัวเลือก B: Accordion ใน Drawer เดิม**
```
▼ ข้อมูลพื้นฐาน   (ชื่อ + คำอธิบาย)
▶ รูปภาพ
▶ ราคา & ส่วนลด
▶ หมวดหมู่ & ประเภท
```

ดีเพราะ: ไม่เปลี่ยน pattern มาก แค่ยุบ section ที่ไม่ได้ใช้

---

### 2.6 Search ที่ฉลาดขึ้น — ค้นข้ามหมวดได้

**ปัจจุบัน:** Search อยู่ใน panel ซ้าย (ค้นหมวดหมู่) และ panel ขวา (ค้นเมนูในหมวด)  
**แก้:** Search global ด้านบนสุด ค้นได้ทั้ง ชื่อเมนู + ชื่อหมวดหมู่

```
🔍  ค้นหาเมนูหรือหมวดหมู่...                    [+ เพิ่มอาหาร]

ผลลัพธ์: "ข้าวผัด"
  ├─ เมนูขายดี  →  ข้าวผัดกุ้ง
  ├─ เมนูใหม่   →  ข้าวผัดทะเล
  └─ เมนูอื่น   →  ข้าวผัดไข่
```

---

### 2.7 Sticky Header + Progress indicator

**ปัจจุบัน:** Header "จัดการเมนู" หายไปเมื่อ scroll  
**แก้:**
- Sticky header แสดงชื่อหมวดที่เลือก + จำนวนเมนู ตลอดเวลา
- แถบด้านล่าง fixed: ปุ่ม "ยกเลิก / บันทึก" อยู่ที่เดิมเสมอ ไม่ต้อง scroll ลงหา

## 4. สรุป Flow การใช้งานหลังปรับ (Happy Path)

```
เปิดหน้า จัดการเมนู
  │
  ├─ เห็น Category ซ้าย → Collapse ทุกหมวด ยกเว้นหมวด active
  │   Badge บอกจำนวนเมนูทันที
  │
  ├─ คลิกหมวดที่ต้องการ → ขวาแสดงเมนู
  │   เลือก List View → เห็น 12-15 รายการต่อหน้า
  │
  ├─ ต้องการแก้ราคา → Double-click ราคา → พิมพ์ใหม่ → Enter
  │
  ├─ ต้องการลบหลายอัน → ติ๊ก checkbox → Bulk delete
  │
  └─ ต้องการเพิ่มเมนู → กด "+ เพิ่มอาหาร" → Step 1 → 2 → 3 → บันทึก
```

## 5. สิ่งที่ดีอยู่แล้ว — อย่าเปลี่ยน

- Drag & Drop reorder หมวดหมู่ (มีแล้วใน design)
- Filter tab (ต้ม/ผัด/แกง...) ช่วย narrow down ได้ดี
- Drawer เพิ่มเมนูจากขวา (ดีกว่า full-page modal)
- Layout 2 panel (Category ซ้าย + เมนูขวา) — intuitive
